# BoC Tracker

A personal data dashboard tracking the economic indicators the Bank of Canada watches between its quarterly Monetary Policy Reports. Daily auto-refresh from public APIs; practitioner-grade analytical commentary in plain language; verified analytical framework anchored to BoC primary sources.

**Live dashboard:** https://jayzhaomurray.github.io/boc-tracker/

## What it does

Six analytical sections, each with charts and a short prose blurb that synthesizes the data against a verified analytical framework:

- **Monetary Policy** — BoC overnight rate vs. neutral band; BoC-Fed spread tiers; 2-year yield expectations; balance sheet phase (QE / reinvestment / passive QT / floor maintenance); meeting-resolution action-state classification (on hold / cutting / hiking) with consecutive-meeting counts.
- **Inflation** — headline + core measures + breadth + sub-aggregates (food, energy, goods, services); inflation expectations (CSCE consumer 1y/5y, BOS firms expecting > 3%); written reads grounded in the BoC's preferred core measures and a four-state breadth classification.
- **GDP & Activity** — monthly real GDP + quarterly Q/Q AR + demand-side contribution decomposition; reads grounded in BoC's potential output estimates and the C.D. Howe Business Cycle Council's recession definition (depth + duration + breadth, not the two-negatives shorthand).
- **Labour Market** — unemployment trajectory + percentile of recent range; four wage measures (LFS, LFS permanent, SEPH, BoC's composition-adjusted LFS-Micro) with dispersion and pass-through-to-services-CPI reads.
- **Housing** — starts (with 3-month moving average for noise filtering), New Housing Price Index, residential permits as leading indicator (multi-unit lag of 9-15 months for the current Canadian housing mix); CMHC supply-gap context.
- **Financial Conditions** — WTI / Brent / WCS oil with implied gasoline-channel CPI impulse; USDCAD with stress-corridor flag (1.45-1.47); BoC FX-pass-through magnitudes; petrocurrency-relationship weakness post-2016.

The dashboard is pure HTML on GitHub Pages — no server, no database, no JavaScript framework. Charts are Plotly, generated by a Python build step and embedded directly into the page.

## Architecture

```
fetch.py     ← pulls daily from StatsCan WDS, BoC Valet, FRED, Alberta Economic Dashboard;
               saves CSVs to data/
analyze.py   ← reads framework + CSVs, calls Claude, writes data/blurbs.json
               (with self-review pass for factual correctness)
build.py     ← reads CSVs + blurbs.json, builds index.html with charts and section blurbs
```

CI runs all three nightly via `.github/workflows/update.yml`; `git-auto-commit-action` pushes the rebuilt dashboard back to `main`, which triggers GitHub Pages redeploy. The cron has three scheduled triggers (two for the StatsCan release window, one for the BoC daily yields update) plus manual dispatch.

## Analytical framework

The dashboard's voice and analytical reads are governed by `markdown-files/analysis_framework.md` — a per-section brief that defines the question being answered, the signals to evaluate, the thresholds that matter, and what to surface in the output. Every analytical claim in the framework is anchored to a BoC primary source (Staff Analytical Notes, MPRs, FSRs, official communications) or empirically verified against our own data. Each section carries an explicit `VERIFICATION STATUS` flag at the top so a reader knows the standing of the claims that drive its blurb.

The framework is the load-bearing document for everything else. The blurbs are downstream of it; the verification audits done in May 2026 are what make the dashboard practitioner-grade rather than a generic data viewer.

## Data sources

- **Statistics Canada WDS** — CPI, employment, wages, GDP, housing starts, NHPI, building permits, and CPI components for the breadth chart
- **Bank of Canada Valet API** — overnight rate (daily V39079 for meeting-resolution analysis + monthly aggregate for chart history), 2Y GoC yield, core inflation measures (trim/median/common/CPIX/CPIXFET), LFS-Micro composition-adjusted wages, balance sheet (weekly), inflation expectations (CSCE/BOS), FAD calendar (iCal feed)
- **Federal Reserve (FRED)** — US 2Y Treasury, USD/CAD exchange rate, WTI / Brent oil, fed funds target
- **Alberta Economic Dashboard** — Western Canada Select crude price

All free, all public, no Bloomberg or Refinitiv. The full data fetch takes ~30-60 seconds; the build is offline and takes a few seconds.

## How it was built

This project was built collaboratively with Claude (Anthropic), under the direction of jayzhaomurray. AI collaboration patterns used throughout:
- **Verification cycles via parallel subagents** — every analytical claim in the framework was audited against BoC primary sources by spinning up multiple Sonnet agents in parallel for primary-source research, while pandas runs locally for empirical distribution work. Findings tracked as VERIFIED / WRONG / NEEDS REFINEMENT before any framework rewrite or blurb generation.
- **Self-review pass on blurb generation** — `analyze.py` makes a second LLM call after each blurb is generated, with a focused factual-correctness checklist (asset/liability, sign/direction, attribution, dates, magnitudes). Catches the failure modes that one-shot prompting misses on domain content.
- **Cross-session memory** — durable principles for analytical writing (plain-language voice, what-over-why, structured-formatter-encodes-semantics, translate-technical-constructs-with-care) live in a memory file that loads in every future session, so the analytical voice stays consistent without re-deriving it.

The git history is the working evidence of how the project evolved — each commit a discrete iteration, with commit messages that explain the *why* of changes. Worth scrolling through if you're evaluating analytical and technical decision-making.
